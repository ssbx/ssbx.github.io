My things

Supercast

The first work on Supercast was around 2011, when the rfc6455 (WebSocket Protocol) was close to become a standard. There was some early implementations of it in Erlang, so I began to learn this nice language. There was not, and in my opinion there is still not today, any libraries/frameworks allowing to efficiently create maintainable applications offering soft real time behaviour to the user interface. Supercast is a solutions to this need. I have used it successfuly in my actual main project Sysmo-NMS.

Sysmo-NMS

It is actually the main project I work on. There is a commercial website. Take a look if you wish :)

NChecks

The work on NChecks began with Sysmo, wile working on nagios-plugins compatibility. Nagios-plugins are popular because they are so simple to write. But I found there are some major drawbacks. NChecks fix problems like portability, possiblity to store a state between checks, many alternative status. It is a major subproject success of the Sysmo thing.

FunLibs

While having worked with lot of programming and scripting langages, I find myself missing some experience in C language. I've started learning and creating some libs. The first were Cargo, a simple argc/argv parser and Clog, a log library. After having created some more libs i decided to create the github funlibs org to host the code in a coherent way. It now include many libs around multimedia oriented applications (games), like playing/mixing sounds with portaudio (LibShake), mathematics for OpenGL (CGLM), fork of some utilities (mostly modified for CMake compat). Most (all) of these libs are used by my Federation pet project.
